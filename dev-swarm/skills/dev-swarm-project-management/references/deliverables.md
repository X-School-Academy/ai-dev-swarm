# 09-sprints deliverables

Only present files that are suitable for the project's type and level; omit the rest.

## Deliverable list

- README.md: Sprint plan overview and cumulative milestones.
- sprint-feature-proposal.md: Proposed sprint sequence and features.
- sprint folders: Per-sprint specs and backlogs.

## File content guidance

### README.md (The Master Plan)

- Keep it simple: several lines indicating project level and master plan
- Owners: Project Manager and Tech Manager
- Strategy: how the product evolves through cumulative demo-able milestones (sprints)
- Timeline: estimated number of sprints to reach MVP
- Diagrams (if required by project init):
  - Reference `dev-swarm/docs/mermaid-diagram-guide.md`
  - Include `diagram/` deliverables when needed

### sprint-feature-proposal.md

- Propose the sequence of sprints
- Each sprint must be a cumulative update to the demo-able product
- List the high-level features for each sprint
