# User Stories

## Stage Management

**As Sarah (PM), I want to** see all development stages at a glance **so that** I know where the project stands without asking anyone.

**As Sarah (PM), I want to** skip stages that aren't relevant to my project **so that** I don't waste time on unnecessary steps.

**As Marcus (BA), I want to** understand what each stage does **so that** I can decide which ones to use for my documentation needs.

**As Alex (Dev), I want to** see which stages are complete, in-progress, or skipped **so that** I can quickly assess project status.

## Document Viewing and Editing

**As Sarah (PM), I want to** view AI-generated documents in a readable format **so that** I can review them without opening separate applications.

**As Sarah (PM), I want to** edit documents directly in the UI **so that** I can make corrections without learning a code editor.

**As Marcus (BA), I want to** see markdown rendered properly **so that** I can focus on content rather than formatting syntax.

**As Alex (Dev), I want to** quickly view any project document **so that** I don't have to navigate folder structures manually.

## AI Execution and Control

**As Sarah (PM), I want to** start a stage with one click **so that** I don't need to remember or type commands.

**As Sarah (PM), I want to** see AI output in real-time **so that** I know what's happening and can identify problems early.

**As Sarah (PM), I want to** stop AI execution if something goes wrong **so that** I don't waste time on bad outputs.

**As Marcus (BA), I want to** approve proposals before AI generates files **so that** I maintain control over what gets created.

**As Alex (Dev), I want to** interrupt AI mid-execution **so that** I can correct course when it misinterprets requirements.

**As Alex (Dev), I want to** see streaming output **so that** I can monitor progress like I would in a terminal.

## Sprint Management (Stage 10)

**As Sarah (PM), I want to** run a backlog with one click **so that** I can see dev, review, and test happen automatically.

**As Sarah (PM), I want to** run dev, review, and test separately **so that** I can review results between each step when needed.

**As Alex (Dev), I want to** run an entire sprint automatically **so that** I can process multiple backlogs without manual intervention.

**As Alex (Dev), I want to** run all sprints in sequence **so that** I can automate the entire implementation phase.

## Project Sync

**As Alex (Dev), I want to** see changes made via terminal reflected in the UI **so that** I can use both interfaces interchangeably.

**As Sarah (PM), I want to** refresh the UI to see the latest project state **so that** I know I'm looking at current information.

**As Marcus (BA), I want to** manually trigger sync **so that** I control when the UI updates and avoid surprises.

## Error Handling

**As Sarah (PM), I want to** see clear error messages **so that** I understand what went wrong without technical knowledge.

**As Sarah (PM), I want to** know what to do when an error occurs **so that** I can recover without developer help.

**As Alex (Dev), I want to** see detailed error information **so that** I can diagnose and fix issues quickly.

## Story Priority for MVP

**Must Have (P0):**
- View all stages with their status
- View and edit markdown documents
- Start stage execution with one click
- See real-time AI output
- Stop AI execution

**Should Have (P1):**
- Skip/unskip stages via UI
- Approve proposals before file generation
- Manual project sync/refresh

**Nice to Have (P2):**
- One-click sprint/backlog automation
- Run all sprints in sequence
- HTML document viewing
