# Stage 02 - Personas

## Stage Goal

This stage creates detailed user personas that bring the target users identified in Stage 00 to life. Personas transform abstract user segments into concrete, relatable characters that guide design and development decisions throughout the project.

User personas are critical because they keep the team focused on real user needs rather than assumptions. For the Dev Swarm WebUI, we have two distinct user groups with different motivations: non-technical stakeholders who need accessibility, and technical users who want efficiency. Clear personas ensure we design for both without sacrificing one for the other.

This stage builds on the target users and user scenarios from `00-init-ideas/target-users.md`, expanding them into actionable personas with user stories that will drive MVP prioritization (Stage 03) and PRD requirements (Stage 05).

## Proposed Files

Based on the project scope (internal tool with two clear user segments), I recommend:

**User Personas**

- `personas.md`
  - 3 detailed personas: 2 primary (non-technical) + 1 secondary (technical)
  - Demographics, background, goals, frustrations, tech comfort level
  - Essential for humanizing the target users and guiding design decisions

**User Stories**

- `user-stories.md`
  - Stories in "As a [persona], I want to [action], so that [benefit]" format
  - Grouped by persona and feature area
  - Essential for defining what the product must do for each user type

**Pain Points**

- `pain-points.md`
  - Detailed pain points mapped to each persona
  - Current workarounds and their costs
  - Essential for ensuring the solution addresses real problems

**Not included:**
- `empathy-maps/` - The personas will capture key insights; separate empathy maps add overhead without proportional value for this internal tool
- `user-journey-maps/` - The workflow is linear (stages 00→11); detailed journey maps are unnecessary
- `use-cases.md` - User stories will serve this purpose sufficiently
