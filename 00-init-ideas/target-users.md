# Target Users

## Primary Users: Non-Technical Stakeholders

**Who they are:**
- Product managers defining requirements
- Business analysts documenting workflows
- Designers contributing to UX stages
- Founders or executives exploring ideas
- Anyone who wants to use AI-driven development without learning CLI

**Their needs:**
- Visual interface they can understand without technical training
- Clear feedback on what's happening and what to do next
- Ability to review and approve AI-generated content
- Control over the process without writing commands

**Their pain with current system:**
- Intimidated by terminal interface
- Don't know the slash commands
- Can't interpret raw AI output
- Feel dependent on developers to operate dev-swarm

## Secondary Users: Technical Users Who Prefer Visual Workflow

**Who they are:**
- Developers who know CLI but prefer visual management
- Team leads overseeing multiple projects
- Users who want better visibility into stage progress
- Anyone debugging or reviewing AI workflow outputs

**Their needs:**
- Quick overview of project state across all stages
- Easy access to view/edit documents without navigating folders
- Ability to interrupt and correct AI operations
- Option to still use terminal when preferred (sync feature)

**Their pain with current system:**
- Mentally tracking which stages are done
- Switching between terminal and file browser
- No easy way to interrupt a running AI agent
- Inconsistent AI behavior requiring re-runs

## User Scenarios

### Scenario 1: Product Manager Starting a New Project

Sarah is a product manager with an idea for an internal tool. She opens the web UI, writes her ideas in the built-in editor, and clicks through the stages. She reviews each proposal, provides feedback, and approves the AI-generated documents—all without touching a terminal.

### Scenario 2: Developer Running Sprint Backlogs

Alex is a developer who prefers clicking through sprint execution rather than typing commands. They use the sprint management UI to run backlogs one by one, watching the real-time output and interrupting when the AI goes off track.

### Scenario 3: Mixed Terminal and UI Usage

Jordan uses Claude Code directly in their terminal for some tasks but switches to the web UI for an overview of stage completion. The UI syncs automatically and shows the current state regardless of how changes were made.
