# Accessibility

## Contrast and Color

- All text must meet WCAG 2.1 AA contrast ratios.
- Status colors must be paired with labels, not color alone.
- Focus states must be visible against all surfaces.

## Keyboard Navigation

- All interactive elements must be reachable via Tab.
- Provide visible focus indicators for buttons, inputs, and links.
- Keyboard shortcuts should be documented and optional.

## Screen Reader Support

- Use semantic HTML elements for layout and navigation.
- Provide ARIA labels for icon-only buttons.
- Output console should use live region labeling for new messages.

## Motion

- Avoid motion that could trigger vestibular issues.
- Provide a reduced-motion option for any animations.
